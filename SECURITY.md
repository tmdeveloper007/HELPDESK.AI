# Security Policy

## Supported Versions

The following versions of HELPDESK.AI are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of HELPDESK.AI very seriously. If you discover a security vulnerability, we kindly ask that you follow this process to report it privately.

Please do **not** report security vulnerabilities through public GitHub issues. 

Instead, please report them via email to `masteradmin@helpdesk.ai` or utilize GitHub's private vulnerability reporting feature.

### What to include in your report
To help us quickly validate and resolve the issue, please include:
- A clear description of the vulnerability and its potential impact.
- Detailed steps to reproduce the vulnerability.
- Any relevant scripts, payloads, or screenshots.
- Your contact information for further communication.

### Our Response Process
We take all security vulnerabilities seriously. Once a vulnerability is submitted, we will:
1. Acknowledge receipt of your report within 48 hours.
2. Strive to send you regular updates about our progress in investigating and mitigating the issue.
3. Release a patch or advisory once the issue has been resolved.

## Out of Scope
The following types of reports are generally out of scope:
- Missing security headers which do not lead directly to a vulnerability.
- Clickjacking on pages with no sensitive actions.
- Self-XSS or XSS that requires significant user interaction.
- Vulnerabilities in third-party dependencies unless directly exploitable in our application.

Thank you for helping us keep HELPDESK.AI safe and secure for all users!
